# タスク: review-quality

## 概要
UI コンポーネントを計画・実装・品質レビューする。

## ワークフローモード
このタスクは Plan → Work → Review の3ステップで評価されます。

## プロンプト

```
ダッシュボードコンポーネントを実装してください。

要件:
- ユーザー統計の表示（アクティブユーザー数、セッション数）
- グラフ表示（過去7日間のトレンド）
- リアルタイム更新機能
- レスポンシブデザイン

技術スタック: TypeScript, React
出力先: src/components/Dashboard.tsx
```

## 期待される出力

### Plan ステップ
- コンポーネント設計が Plans.md に追加される

### Work ステップ
- `src/components/Dashboard.tsx` が作成される
- 統計表示・グラフ機能が実装される

### Review ステップ
- 品質問題が Severity 付きで出力される
- 重点項目: 可読性, パフォーマンス, ベストプラクティス, 保守性

## 成功基準
| 基準 | 条件 |
|------|------|
| ファイル作成 | Dashboard.tsx が存在 |
| React コンポーネント | 関数コンポーネントが定義 |
| レビュー出力 | Severity 付きの指摘がある |

## 測定ポイント
- 全ステップの合計時間
- 検出された品質問題数
- 改善提案の具体性
