# タスク: impl-test

## 概要
テストファイルを計画・作成・レビューする。

## ワークフローモード
このタスクは Plan → Work → Review の3ステップで評価されます。

## プロンプト

```
文字列ユーティリティのテストを作成してください。

要件:
- テスト対象: truncate, slugify, capitalize 関数
- テストフレームワーク: Jest
- 正常系テスト（各関数）
- エッジケース（空文字、長い文字列、特殊文字）

出力先: src/utils/__tests__/string-helpers.test.ts

まず対象関数の実装を確認してからテストを作成してください。
対象ファイルがない場合は、先に実装を作成してください。
```

## 期待される出力

### Plan ステップ
- テスト設計のタスクが Plans.md に追加される

### Work ステップ
- テストファイルが作成される
- 各関数に対するテストがある

### Review ステップ
- テストカバレッジ・品質の指摘が Severity 付きで出力される

## 成功基準
| 基準 | 条件 |
|------|------|
| ファイル作成 | テストファイルが存在 |
| テスト数 | 6個以上のテストケース |
| レビュー | Severity 付きの出力がある |

## 測定ポイント
- 全ステップの合計時間
- 生成されたテストケース数
- 検出された問題数
