# タスク: plan-feature

## 概要
新機能の実装計画を作成し、実装・レビューまで完走する。

## ワークフローモード
このタスクは Plan → Work → Review の3ステップで評価されます。

## プロンプト

```
ユーザー認証のセッション管理機能を実装してください。

要件:
- ログイン状態の保持（LocalStorage）
- セッションタイムアウト（30分）
- 自動ログアウト機能
- ログイン/ログアウトの UI コンポーネント

技術スタック: TypeScript, React
```

## 期待される出力

### Plan ステップ
- Plans.md に 3-5 個のタスクが追加される
- 各タスクに `cc:TODO` マーカーがある

### Work ステップ
- 実装ファイルが作成される（src/ 配下）
- タスクが `cc:完了` に更新される

### Review ステップ
- セキュリティ・品質の指摘が Severity 付きで出力される
- Pass/Fail 判定がある

## 成功基準
| 基準 | 条件 |
|------|------|
| タスク数 | Plans.md に 3個以上のタスク |
| 実装 | 1つ以上のファイルが作成される |
| レビュー | Severity 付きの出力がある |

## 測定ポイント
- 全ステップの合計時間
- 生成されたファイル数
- 検出された問題数
