# Extended Thinking ガイド

`claude-code-harness`は、複雑なタスクを処理する際に**Extended Thinking**を活用することで、より高品質な成果を生み出します。

## 1. Extended Thinkingとは？

Extended Thinkingは、Claudeが複雑な問題に対して、即座に結論を出すのではなく、思考プロセスを明示的に書き出しながら段階的に解決策を導き出す機能です。

これにより、以下のようなメリットがあります。

- **思考の透明性**: Claudeが何を考えているかが明確になり、ユーザーはプロセスを理解・修正できます。
- **品質の向上**: 複数の選択肢を検討し、最適な解決策を選択するため、最終的な成果物の品質が向上します。
- **エラーの削減**: 早計な結論を避け、慎重に検討することで、エラーや見落としを減らします。

## 2. Extended Thinkingのトリガー

以下のキーワードやフレーズをプロンプトに含めることで、Extended Thinkingを意図的にトリガーできます。

### 推奨トリガーフレーズ

- **「ステップバイステップで考えて」**: 段階的な思考を促します。
- **「複数の選択肢を検討して」**: 最適な解決策を探させます。
- **「トレードオフを評価して」**: 各選択肢の長所と短所を比較させます。
- **「潜在的な問題を洗い出して」**: リスクを事前に特定させます。
- **「最初に計画を立ててから実行して」**: 実行前の計画策定を強制します。

## 3. `claude-code-harness`での活用例

### `/plan`コマンド

```
/plan "新しい認証システムを設計する。ステップバイステップで考えて、複数の選択肢（JWT, OAuth, SAML）のトレードオフを評価してください。"
```

### `/work`コマンド

```
/work "データベースのスキーマを変更する。潜在的な問題を洗い出し、安全なマイグレーション手順を計画してから実行してください。"
```

### `/review`コマンド

```
/review "このコードのパフォーマンスをレビューしてください。複数の改善案を検討し、最も効果的なものを提案してください。"
```

## 4. ベストプラクティス

- **複雑なタスクで活用**: 簡単なタスクでは不要ですが、設計、リファクタリング、大規模な変更など、複雑なタスクで特に有効です。
- **具体的な指示を与える**: 単に「考えて」と言うだけでなく、何をどのように考えてほしいかを具体的に指示します。
- **思考プロセスをレビュー**: Claudeが書き出した思考プロセスを確認し、必要に応じて軌道修正を行います。

Extended Thinkingを積極的に活用することで、`claude-code-harness`の能力を最大限に引き出し、より高品質な開発を実現できます。
