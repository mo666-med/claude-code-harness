# /start-task - タスク開始

Cursor（PM）から受け取ったタスクを開始するためのセットアップを行います。

---

## 実行フロー

### Step 1: 現在の状態を確認

以下を並列で実行してください：

```bash
# Git状態
git status -sb

# 最近のコミット
git log --oneline -5
```

```bash
# Plans.md の内容確認
cat Plans.md
```

```bash
# AGENTS.md の確認（役割分担の把握）
head -100 AGENTS.md
```

### Step 2: 進行中タスクの特定

`Plans.md` から以下のマーカーを探してください：

| マーカー | 意味 |
|---------|------|
| `cc:WIP` | Claude Code が作業中 |
| `cc:TODO` | Claude Code に割り当て済み、未着手 |
| `cursor:依頼中` | Cursor から依頼されたタスク |

**優先度**: `cursor:依頼中` > `cc:TODO` > `cc:WIP`（既存作業の継続）

### Step 3: タスク内容の確認と報告

見つかったタスクをユーザーに報告：

> 📋 **本日のタスク**
>
> **メインタスク**: {{タスク名}}
> - 概要: {{タスクの説明}}
> - スコープ: {{変更対象ファイル・範囲}}
> - 完了条件: {{明確な受け入れ基準}}
>
> **依存タスク**（あれば）:
> - {{依存するタスク}}
>
> **作業を開始しますか？** (y/n)

### Step 4: TodoWrite でタスク分解

ユーザーが承認したら、TodoWrite ツールを使ってタスクを分解：

```
例:
1. 関連コードの調査
2. 実装
3. テスト実行
4. コミット & プッシュ
5. CI通過確認
```

### Step 5: Plans.md の更新

タスクのマーカーを更新：

```markdown
# 変更前
- [ ] タスク名 `cursor:依頼中`

# 変更後
- [ ] タスク名 `cc:WIP`
```

### Step 6: 作業開始の宣言

> 🚀 **作業開始**
>
> タスク「{{タスク名}}」の作業を開始します。
>
> 完了したら `/handoff-to-cursor` で報告します。

---

## タスクが見つからない場合

> ⚠️ **進行中のタスクが見つかりません**
>
> 以下を確認してください：
> 1. Cursor で `/handoff-to-claude` を実行してタスクを依頼
> 2. `Plans.md` に直接タスクを追加
>
> または、やりたいことを教えてください。

---

## 注意事項

- **AGENTS.md を必ず確認**: 禁止事項や役割分担を把握
- **Plans.md を更新**: 作業状態を常に反映
- **スコープを守る**: 依頼された範囲外の作業は行わない
